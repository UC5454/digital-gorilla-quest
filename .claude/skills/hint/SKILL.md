# /hint - ヒント表示

先輩ゴリラからのアドバイスを表示する。

## 起動時の動作

1. `gorilla_data/progress.json` を読み込む
2. 現在のクエスト（current）を特定
3. クエストファイルからヒントセクションを読み込む
4. 段階的にヒントを表示

## ヒントの段階

各クエストには3段階のヒントを用意：

1. **ヒント1（軽いアドバイス）**: 方向性を示す
2. **ヒント2（具体的なヒント）**: 具体的な手順やコマンドの一部
3. **ヒント3（ほぼ答え）**: 直接的な解答に近い情報

## 表示フォーマット

### 先輩ゴリラ登場
```
🦍💬 先輩ゴリラの助言 💬🦍
════════════════════════════════

  ウホッ！困っているのか？

  俺も昔は苦労したもんだ...
  よし、教えてやろう！

════════════════════════════════
```

### ヒント1（初回）
```
  【ヒント Lv.1】

  [ヒント内容]

  💡 まだ分からなかったら、もう一度
     /hint を呼んでくれ！

════════════════════════════════
```

### ヒント2（2回目）
```
  【ヒント Lv.2】

  ウホホ、まだ難しいか？
  もう少し詳しく教えてやろう...

  [より具体的なヒント]

  💡 これでもダメなら、最後の手段だ...

════════════════════════════════
```

### ヒント3（3回目）
```
  【ヒント Lv.3 - 最終ヒント】

  ウホ...仕方ないな。
  ほぼ答えを教えてやる。
  でもちゃんと理解するんだぞ！

  [ほぼ答え]

════════════════════════════════
  📚 関連リファレンス: docs/reference/[xx].md
════════════════════════════════
```

## ヒントカウントの管理

progress.jsonにヒントカウントを保持し、同じクエストで /hint を呼ぶたびにインクリメント。
クエストが変わったらリセット。

## クエストが設定されていない場合

```
🦍❓ 先輩ゴリラ ❓🦍
════════════════════════════════

  ウホ？何について聞きたいんだ？

  まずは /quest [番号] でクエストを
  開始してから来てくれ！

════════════════════════════════
```
