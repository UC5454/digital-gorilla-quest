# /quest - クエスト開始

指定されたクエストの内容を表示し、冒険を開始する。

## 使用方法

```
/quest [番号]
```

番号を省略した場合は、現在のエリアで次にクリアすべきクエストを表示。

## 起動時の動作

1. `gorilla_data/progress.json` を読み込む
2. `quest_file_map` からクエスト番号に対応するファイルパスを取得
3. `quests/[ファイルパス]` のクエストファイルを読み込む
4. クエスト内容を演出付きで表示
5. progress.jsonのcurrent questを更新

## クエストファイルの場所

progress.json の `quest_file_map` を参照してファイルパスを特定する。
全クエストは `quests/` ディレクトリ配下にエリア別フォルダで格納。

| クエスト番号 | エリア | ディレクトリ |
|-------------|--------|-------------|
| Q1-Q6 | 草原 | quests/grassland/ |
| Q7-Q12 | 森林 | quests/forest/ |
| Q13-Q18 | 山岳 | quests/mountain/ |
| Q19-Q24 | 洞窟 | quests/cave/ |
| Q25-Q30 | 神殿 | quests/temple/ |
| Q31-Q36 | 祭壇 | quests/altar/ |
| Q37-Q42 | 砦 | quests/fortress/ |
| Q43-Q48 | 火山 | quests/volcano/ |
| Q49-Q54 | 頂上 | quests/summit/ |

## 表示フォーマット

```
🦍🌴 QUEST [番号] 🌴🦍
================================

  📍 [エリア名] - [クエスト名]

  [クエストの導入テキスト（ゴリラ口調）]

  【目標】
  [目標リスト]

  【成功条件】
  [条件リスト]

  【報酬】
  🍌 [BP] BP
  [バッジがある場合: 🥇 バッジ「XXX」]

================================
  💡 困ったら /hint で先輩ゴリラに聞け！
  ✅ 完了したら /banana で報告！（または /go done）
```

## バリデーション

- 未解放エリアのクエストは開始不可
- 前のクエストが未完了の場合は警告を表示（ただし開始は可能）
- 既にクリア済みのクエストは再挑戦として表示
