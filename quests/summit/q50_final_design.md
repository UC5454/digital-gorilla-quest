## 💡 ヒント

### hint_1
skillsセクションには「何を、どの順番で、どう処理するか」を具体的に書く。曖昧な指示よりも、明確な手順書の方がAI社員は正確に動く。

### hint_2
良いskills記述の例:
```
## 日次レポート作成スキル
1. daily-logs/ から今日の全ログを読み込む
2. 各ログのハイライトを抽出する
3. カテゴリ別に整理する
4. outputs/daily-report-[日付].md に保存する
5. 完了したらdaily-logsに実行記録を追記する
```

### hint_3
CLAUDE.md最終チェックリスト:
- [ ] identity: 名前、役割、一人称、性格が定義されている
- [ ] role: 責任範囲、判断基準、得意/不得意が明確
- [ ] persona: コミュニケーションスタイル、口調の例がある
- [ ] skills: 業務フロー用のスキルが具体的に記述されている
- [ ] startup: 起動時の読み込みファイル・確認事項がリストされている
- [ ] inbox_rules: メッセージの送受信ルールが定義されている
- [ ] 全体で150行以上ある
