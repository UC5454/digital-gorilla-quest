# Quest 28: 守りの心得

## 導入テキスト

ウホッ！流れを作れるようになったな。

だが、強さだけでは足りない。
「守りの心得」も必要だ。

セキュリティを意識した Claude Code の使い方を学べ！

## 目標

- Claude Code 使用時のセキュリティリスクを理解する
- 安全に使うためのベストプラクティスを把握する

## 成功条件

- [ ] やってはいけないこと（APIキーの露出など）を3つ以上挙げられる
- [ ] 安全に使うためのベストプラクティスを理解した

## 報酬

- base_bp: 700
- first_clear_bonus: 140

## ヒント

### hint_1
絶対にやってはいけないこと：
- APIキーを CLAUDE.md に書く
- 本番DBに直接接続
- rm -rf などの危険コマンドを自動化
- パスワードをコードにハードコード

### hint_2
安全に使うためのベストプラクティス：
1. 環境変数を使う
2. サンドボックス環境でテスト
3. 重要操作前に確認を入れる
4. .gitignore で機密ファイルを除外

### hint_3
セキュリティチェックリスト：

**やってはいけないこと**:
- [ ] APIキーをソースコードに含めない
- [ ] 本番環境で直接 Claude Code を実行しない
- [ ] 機密情報を CLAUDE.md に書かない
- [ ] 危険なコマンドを自動化しない

**やるべきこと**:
- [ ] 環境変数で機密情報を管理
- [ ] .gitignore を適切に設定
- [ ] PreToolUse フックで危険操作を検知
- [ ] 定期的に設定を見直す
