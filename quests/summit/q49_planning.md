# Quest 49: 王の構想

## 🦍 導入

ウホッ！ついに頂上だ...ここまで来たお前は、もう立派なゴリラだ。
だが、最後のエリアは「技術」ではなく「構想」が問われる。

どんなに強い技を持っていても、使い道がなければ意味がない。
お前のAI社員を、実際の業務でどう活かすか——王の構想を描け！

## 🎯 目標

- [ ] AI社員の活用シーンを3つ以上リストアップする
- [ ] 1つを選定し、業務フローを設計する
- [ ] 設計を `my-first-ai/docs/use-case-design.md` に保存する

## 📋 やること

1. 自分の日常業務や興味のある領域から、AI社員が活躍できるシーンを3つ以上考える
2. 最も効果が高そうな1つを選ぶ
3. その活用シーンの業務フロー（入力→処理→出力）を設計する
4. `my-first-ai/docs/use-case-design.md` に保存する

## 📎 コピペ

```
以下を考えて my-first-ai/docs/use-case-design.md に保存して：

## 活用シーン候補

1. [シーン1: 例「毎朝のニュース要約」]
2. [シーン2: 例「議事録の自動整理」]
3. [シーン3: 例「ブログ記事の下書き作成」]

## 選定したシーン

[最も効果が高いものを1つ選ぶ]

選定理由: [なぜこれを選んだか]

## 業務フロー設計

### トリガー（いつ・何がきっかけで動くか）
[例: 毎朝9時、またはINBOXにメッセージが来たら]

### 入力（何を受け取るか）
[例: ニュースのURL一覧、会議の音声テキスト]

### 処理（AI社員が何をするか）
[例: 要約→構造化→優先度付け]

### 出力（何を生成するか）
[例: daily-report.md に保存、INBOXに報告]

### 成功基準（どうなったら成功か）
[例: 5分以内に完了、人間のチェックなしで使える品質]
```

## 💡 ヒント

### hint_1
活用シーンは「毎日やっていて面倒なこと」「やりたいけど時間がなくてできていないこと」から考えると見つかりやすい。

### hint_2
業務フローは5W1H（いつ、誰が、何を、どこで、なぜ、どうやって）で整理すると漏れがない。特に「トリガー（いつ動くか）」と「出力（何が生まれるか）」を明確にすることが重要。

### hint_3
活用シーンの例:
- **リサーチ系**: 毎日の業界ニュース要約、競合調査、技術トレンド追跡
- **ライティング系**: ブログ下書き、SNS投稿案、メール返信ドラフト
- **整理系**: 議事録整理、タスク抽出、ドキュメント更新
- **分析系**: データ集計、レポート作成、KPIトラッキング

業務フローの核心は「人間が考える部分」と「AI社員に任せる部分」の境界線を引くこと。

## 📚 学習ポイント

- AI社員の真価は「日常業務への組み込み」で発揮される
- 活用シーンの設計は、技術力より「業務理解」が重要
- 業務フローを明確にすることで、CLAUDE.mdの設計精度が上がる
