# Quest 38: 伝令の仕組み

## 🦍 導入

ウホッ！群れができたら、次は「伝令」の仕組みだ！
仲間同士が情報をやり取りできなければ、群れはただの烏合の衆だ。

INBOX.md ——これがAI社員間の「伝令システム」だ。
メッセージの書き方、読み方、処理の仕方を学べ！

## 🎯 目標

- [ ] INBOX.mdの役割と構造を理解する
- [ ] AI社員用のINBOX.mdを作成する
- [ ] CLAUDE.mdにinbox_rulesセクションを追加する

## 📋 やること

1. AI社員のディレクトリにINBOX.mdを作成する
2. メッセージフォーマット（誰から・いつ・内容）を定義する
3. CLAUDE.mdにinbox_rulesセクションを追加する
4. テストメッセージを書き込んでフォーマットを確認する

## 📎 コピペ

```
my-first-ai/[AI社員ディレクトリ]/INBOX.md を以下の内容で作成して：

# INBOX

## 未処理

（新しいメッセージがここに追記される）

## 処理済み

（処理が完了したメッセージは✅をつけてここに移動）
```

CLAUDE.mdへの追記:

```
my-first-ai/[AI社員ディレクトリ]/CLAUDE.md の末尾に以下を追記して：

<inbox_rules>
## 連携ルール

### メッセージの送り方
他のメンバーに連絡するときは、相手のINBOX.mdの「未処理」セクションに追記する。

書式: 【[自分の名前]より】メッセージ内容

### 自分のINBOXの確認
起動時に自分のINBOX.mdを確認する。
処理したメッセージは「✅完了」をつけて「処理済み」セクションに移動する。
</inbox_rules>
```

## 💡 ヒント

### hint_1
INBOX.mdは「社内チャット」のようなもの。AI社員同士がテキストファイルを通じて非同期にメッセージをやり取りする仕組みだ。

### hint_2
メッセージフォーマットは「誰から来たか」がわかることが大事。【送信者名より】をプレフィックスにすると、複数人からのメッセージが混在しても識別できる。

### hint_3
INBOX.mdのポイントは3つ:
1. 「未処理」セクション — 新着メッセージ置き場
2. 「処理済み」セクション — 完了したメッセージのアーカイブ
3. ✅マーク — 処理完了の目印

CLAUDE.mdのinbox_rulesで「起動時にINBOXを確認する」と書いておけば、AI社員は毎回自動でメッセージをチェックする。

## 📚 学習ポイント

- INBOX.mdはAI社員間の非同期メッセージシステム
- 「未処理→処理済み」のフローで、メッセージの取りこぼしを防ぐ
- CLAUDE.mdにルールを書けば、AI社員は自律的にINBOXを確認する
