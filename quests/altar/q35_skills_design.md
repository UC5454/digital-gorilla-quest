# Quest 35: スキルの設計 skills

## 🦍 導入

ウホッ！個性も決まった。次は「スキル」だ。
AI社員の「得意なこと」と「不得意なこと」を明確にする。
得意なことは積極的に任せ、不得意なことは他に頼む。
さらに、山岳で学んだSkillsを移植して、実用的な技も持たせる！
最強の武器を授けろ！ウホホ！

## 🎯 目標

- [ ] 得意なこと3つ以上、不得意なこと2つ以上を定義する
- [ ] エリア3（山岳）で作ったSkillを移植する
- [ ] .claude/skills/ にスキルファイルを配置する

## 📋 やること

1. AI社員の得意・不得意を定義する
2. 山岳エリアで作成したスキルを移植する
3. `my-first-ai/CLAUDE.md` に skills セクションを追加する

## 📎 コピペ

```
以下を実行して：

1. my-first-ai/CLAUDE.md を読んで、skills セクションを追加して：

<skills>

<capability>
## 得意なこと
- [得意1の説明]
- [得意2の説明]
- [得意3の説明]

## 不得意なこと
- [不得意1の説明]
- [不得意2の説明]
</capability>

## スキル一覧
- /[スキル名1]: [説明]
- /[スキル名2]: [説明]

</skills>

2. practice/mountain/ にあるスキルファイルを my-first-ai/.claude/commands/ にコピーして。
   もしなければ、新しくスキルファイルを1つ作って。

3. スキルファイルの形式:
   my-first-ai/.claude/commands/[スキル名].md
   ```
   ---
   description: [スキルの説明]
   ---
   [スキルの指示内容]
   ```
```

## 💡 ヒント

### hint_1
「得意なこと」は積極的にAI社員に任せる領域、「不得意なこと」は人間がやるか他のAI社員に任せる領域だ。この区分が明確なほど、適材適所の仕事分配ができる。

### hint_2
得意・不得意の例（リサーチャーの場合）:

**得意なこと:**
- Web検索と情報の構造化（複数ソースから要点を抽出）
- ファクトチェック（情報の正確性を複数ソースで検証）
- レポート作成（調査結果を読みやすいMarkdownにまとめる）

**不得意なこと:**
- 最終的な意思決定（判断材料は出すが、決めるのは人間）
- 創造的なアイデア発案（支援はできるが、ゼロから生み出すのは苦手）

### hint_3
スキルファイルの例:
```markdown
---
description: 指定トピックについてWebリサーチし、結果をまとめる
---
以下のトピックについてリサーチしてください：$ARGUMENTS

手順：
1. Fetch MCPで関連情報を3つ以上収集
2. 重要なポイントを箇条書きで整理
3. research-results/ に結果を保存
```

## 📚 学習ポイント

- 得意・不得意を明確にすることで、AI社員の活用範囲が明確になる
- Skills（/コマンド）は繰り返し使う処理を標準化したもの
- .claude/commands/ にスキルファイルを置くと、/コマンドで呼び出せる
