# Quest 18: 技の極意

## 導入テキスト

ウホホホ！お前の技は相当なレベルに達した！

最後の試練だ。
複数のスキルを連携させたり、
エラーハンドリングを含む高度なスキルを作れ。

これができれば、お前は「技の達人」だ！

## 目標

- 高度なスキルを作成する（エラーハンドリング、他スキルとの連携など）
- スキル設計のベストプラクティスを理解する

## 成功条件

- [ ] エラー時の動作を含むスキルを作成した
- [ ] スキルのベストプラクティス（単一責任、明確な命名など）を説明できる

## 報酬

- base_bp: 600
- first_clear_bonus: 120
- badge: "技の達人"
- badge_icon: "🥋"
- badge_description: "スキル作成を完全に習得した証"

## ヒント

### hint_1
エラーハンドリングの例：
```markdown
## エラー時の動作

- ファイルが見つからない: 「ファイルがありません」と表示
- 権限エラー: 「権限がありません」と表示
- その他のエラー: エラー内容を表示
```

### hint_2
ベストプラクティス：
1. 単一責任: 1スキル = 1目的
2. 明確な命名: 何をするか分かる名前
3. 確認を入れる: 破壊的操作前は確認
4. エラーハンドリング: 失敗時の動作を定義

### hint_3
高度なスキル例 - /release：

```markdown
# /release - リリース準備を行う

リリース前のチェックと準備を自動化。

## 起動時の動作

1. 変更がないか確認（git status）
2. テストを実行
3. リントを実行
4. 全て通過したら確認を求める
5. 確認後、バージョンを上げる

## エラー時の動作

- 未コミットの変更がある: 警告を表示し、続行するか確認
- テスト失敗: 失敗したテストを表示し、中止
- リントエラー: エラー内容を表示し、自動修正を提案

## 確認ポイント

- バージョン番号の確認
- CHANGELOG の更新確認
- 最終確認

## 出力形式

🚀 リリース準備
━━━━━━━━━━━━━━━━━━
✅ Git: クリーン
✅ テスト: 全パス
✅ リント: エラーなし
━━━━━━━━━━━━━━━━━━
バージョン: 1.0.0 → 1.1.0
続行しますか？ (y/n)
```
