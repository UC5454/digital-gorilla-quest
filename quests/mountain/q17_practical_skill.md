### hint_1

### hint_2
/todo スキルの例：

```markdown
# /todo - TODOを管理する

プロジェクトのTODOリストを管理。

## パラメータ

- `/todo` - 一覧表示
- `/todo add [内容]` - 追加
- `/todo done [番号]` - 完了

## 起動時の動作

1. TODO.md を読み込む（なければ作成）
2. パラメータに応じた処理
3. 結果を表示
```

### hint_3
practice/skill-dojo/.claude/skills/todo/SKILL.md を作成：

```markdown
# /todo - TODOリストを管理する

シンプルなTODO管理スキル。

## ファイル

TODO.md に保存

## パラメータ

- `/todo` - 一覧表示
- `/todo add [内容]` - 新規追加
- `/todo done [番号]` - 完了マーク
- `/todo clear` - 完了済みを削除

## 起動時の動作

1. TODO.md を読み込む（なければ作成）
2. パラメータを解析
3. 該当する操作を実行
4. 結果を表示

## 出力形式

📝 TODOリスト
1. [ ] タスク1
2. [x] タスク2（完了）
3. [ ] タスク3

## 使用例

/todo add ドキュメントを書く
→ ✅ 追加しました: ドキュメントを書く

/todo done 1
→ ✅ 完了にしました: タスク1
```
