## ⚔️ チャレンジ

オリジナルのワークフロースキルを設計・作成し、その力を証明せよ！

### 1. オリジナルスキルの企画
自分の作業を振り返り、自動化したいワークフローを1つ考えろ。

### 2. スキル作成
`practice/boss-challenge/mountain/` にスキルを作成せよ。

```
practice/boss-challenge/mountain/.claude/commands/ に、
オリジナルのワークフロースキルを作成して。

要件:
- スキル名は自由（自分の作業を自動化するもの）
- 手順は5ステップ以上
- $ARGUMENTS でパラメータを受け取る
- エラーハンドリングを含む
- 出力形式が整理されている

例: /project-init（プロジェクト初期化）、/review（コードレビュー）、
    /search（プロジェクト内検索＋まとめ）、/report（レポート生成）等
```

### 3. 動作テスト
作成したスキルの動作をシミュレーションし、以下を確認せよ。

```
作成したスキルファイルを読んで、以下を検証して：
1. 手順は5ステップ以上あるか？
2. パラメータを正しく受け取れる設計か？
3. エラー時の動作が定義されているか？
4. 出力形式は見やすいか？
5. 改善点があれば指摘して
```

### 4. 改善レポート
検証結果を基にスキルを改善し、最終版を保存せよ。

```
practice/boss-challenge/mountain/skill-review.md を作成して、
以下をまとめて：
1. 作成したスキルの名前と目的
2. 設計のポイント（なぜこの構造にしたか）
3. エラーハンドリングの考慮点
4. 今後の改善案
```
