# Quest 9: ルールの力

## 🦍 導入

ウホッ！CLAUDE.mdを作れるようになったか！
だが、AI社員に「性格」を与えただけじゃ不十分だ。

次に必要なのは「ルール」の力だ。
CLAUDE.mdにルールを書き込めば、AIの行動を正確にコントロールできる。

例えば「コードは必ずTypeScriptで書け」と設定すれば、AIはそのルールに従う。
ルールの力を体験してみろ！

## 🎯 目標

- [ ] CLAUDE.mdにコーディングルールを設定する
- [ ] 設定したルールが実際にClaude Codeの振る舞いを変えることを検証する
- [ ] ルール設定の威力を体感する

## 📋 やること

1. 以下のコピペでルール付きCLAUDE.mdを作成する
2. ルールに従って動作するか検証する指示を出す
3. ルールが効いていることを確認する

## 📎 コピペ

```
practice/forest/rules-test/CLAUDE.md を作成して。以下の内容で：

# ルールテスト用設定

## コーディングルール
- コードは必ずTypeScriptで書くこと
- 変数名はキャメルケース（camelCase）を使うこと
- 全ての関数にJSDocコメントをつけること
- console.logの代わりにconsole.infoを使うこと

その後、practice/forest/rules-test/ にいる前提で「簡単なToDoリストの関数を書いて」と頼んでみて。ルールが守られているか確認して。
```

## ✅ 成功条件

- [ ] コーディングルールが4つ以上設定されている
- [ ] ルールに従ったコードが生成されることを確認した（TypeScript、camelCase等）

## 💡 ヒント

### hint_1
CLAUDE.mdに「こうしろ」と書けば、Claude Codeはそのルールに従うぞ。コーディングスタイル、言語、命名規則、なんでも設定できる。

### hint_2
ルールの書き方は自由だが、箇条書きが効果的だ。「- 必ず〜する」「- 〜を使うこと」のように明確に書こう。曖昧な表現より具体的な指示が効く。

### hint_3
コピペの前半でCLAUDE.mdを作成し、後半で検証を行う。生成されたコードが「TypeScriptで書かれているか」「camelCaseか」「JSDocがあるか」「console.infoを使っているか」を確認しよう。

## 📚 学習ポイント

- **ルールの力**: CLAUDE.mdにルールを書けば、AIの行動を正確にコントロールできる
- **コーディングルール**: 言語・命名規則・スタイル等をプロジェクト単位で統一できる
- **明確な指示**: 具体的で箇条書きのルールが最も効果的
