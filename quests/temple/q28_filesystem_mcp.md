## 💡 ヒント

### hint_1
Claude Codeの標準Readツールは「今開いているプロジェクト」の中だけを読む。ファイルシステムMCPは「指定したディレクトリ」にアクセスできる。つまり活動範囲が広がる！

### hint_2
ファイルシステムMCPの設定例:
```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@anthropic/mcp-server-filesystem",
        "/Users/username/Documents"
      ]
    }
  }
}
```
argsの最後にアクセスを許可するディレクトリパスを指定する。

### hint_3
使い分けの判断:
- プロジェクト内のファイル → 標準Readツール（速い、制限なし）
- プロジェクト外のファイル → ファイルシステムMCP（設定が必要だがアクセス範囲を広げられる）
- セキュリティ: argsで許可するディレクトリを限定する。`/` を指定するとPC全体にアクセスできてしまうので危険！

## 📚 学習ポイント

- 標準ツール = プロジェクト内のみ、ファイルシステムMCP = 指定ディレクトリにアクセス可能
- argsでアクセス範囲を制限する（最小権限の原則）
- セキュリティ: 必要なディレクトリだけを許可し、不要なアクセスは与えない
