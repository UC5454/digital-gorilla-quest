# 📚 CLAUDE.md リファレンス

## CLAUDE.md とは

CLAUDE.md は、Claude Code がプロジェクトのコンテキストを理解するための設定ファイルです。
プロジェクトのルール、スタイルガイド、重要な情報を記述します。

## ファイルの種類と優先順位

| ファイル | 場所 | 用途 |
|----------|------|------|
| `CLAUDE.md` | プロジェクトルート | プロジェクト全体の設定 |
| `CLAUDE.local.md` | プロジェクトルート | 個人設定（.gitignore推奨）|
| `~/.claude/CLAUDE.md` | ホームディレクトリ | グローバル設定 |

読み込み順: グローバル → プロジェクト → ローカル（後が優先）

## 基本構造

```markdown
# プロジェクト名

## 概要
このプロジェクトの説明

## 技術スタック
- 使用言語: TypeScript
- フレームワーク: React
- パッケージマネージャー: npm

## コーディングルール
- インデント: スペース2つ
- 命名規則: camelCase
- コメント: 日本語OK

## 重要なファイル
- `src/index.ts` - エントリーポイント
- `config/` - 設定ファイル群

## 禁止事項
- console.log をコミットしない
- any型を使わない

## コマンド
- `npm run dev` - 開発サーバー起動
- `npm run build` - ビルド
- `npm test` - テスト実行
```

## セクション例

### プロジェクト情報

```markdown
## プロジェクト情報

- 名前: Digital Gorilla App
- バージョン: 1.0.0
- 作者: デジタルゴリラチーム
```

### コーディング規約

```markdown
## コーディング規約

### 命名規則
- 変数・関数: camelCase
- クラス: PascalCase
- 定数: UPPER_SNAKE_CASE

### ファイル構成
- コンポーネント: `components/`
- ユーティリティ: `utils/`
- 型定義: `types/`
```

### 禁止事項・注意事項

```markdown
## 注意事項

### やってはいけないこと
- 本番DBに直接接続しない
- 秘密鍵をコードに含めない
- テストなしでマージしない

### 確認必須
- API変更時はドキュメント更新
- 破壊的変更はチームに相談
```

### カスタムコマンド

```markdown
## カスタムコマンド

このプロジェクトでは以下のスキルが使用可能：
- `/format` - コード整形
- `/test` - テスト実行
- `/deploy` - デプロイ
```

```markdown
形式: `[type]: 概要`
- feat: 新機能
- fix: バグ修正
- docs: ドキュメント
- refactor: リファクタリング
```

## ベストプラクティス

### 1. 簡潔に書く

Claude Code は全文を読むので、冗長な説明は避ける。

✅ 良い例:
```markdown
## コマンド
- `npm run dev` - 開発サーバー
```

❌ 悪い例:
```markdown
## コマンド
開発サーバーを起動するには、ターミナルを開いて、
プロジェクトのルートディレクトリに移動してから、
npm run dev というコマンドを入力してEnterキーを押すと
開発サーバーが起動します...
```

### 2. 具体的に書く

曖昧な表現より、具体的な指示を。

✅ 良い例:
```markdown
- インデント: スペース2つ
- 行末セミコロン: なし
```

❌ 悪い例:
```markdown
- きれいなコードを書く
- 読みやすくする
```

### 3. 更新し続ける

プロジェクトの変化に合わせて CLAUDE.md も更新する。

## トラブルシューティング

**Q: CLAUDE.md が読み込まれない**
A: ファイル名が正確か確認（大文字小文字も重要）

**Q: 設定が反映されない**
A: セッションを再起動するか `/clear` で履歴クリア

**Q: 複数の CLAUDE.md がある場合**
A: 最も深い階層のものが優先される
